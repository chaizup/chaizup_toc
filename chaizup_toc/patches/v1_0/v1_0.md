# `patches/v1_0/` — Version 1.0 Patches

These patches fix issues from the initial v1.0 release. All are idempotent (safe to run multiple times).

---

## `fix_date_filters.py`

### Problem
Number Cards and Dashboard Charts were initially created with date filters using `"Today"` as a raw string. Frappe v14+ requires `"Timespan","today"` format for date filters. This caused Number Cards to show 0 or error.

### Fix
String-replaces the `filters_json` field on:
- Number Cards: `TOC - Items in Red Zone`, `TOC - Items in Yellow Zone`, `TOC - Items in Green Zone`
- Dashboard Chart: `TOC - Zone Distribution`

Replacements:
- `"log_date","=","Today"` → `"log_date","Timespan","today"`
- `"log_date",">=","Today"` → `"log_date","Timespan","today"`

Note: This patch is now partly superseded by `setup/install.py`'s force-recreate logic, which creates the cards fresh using `type="Custom"` + Python `method=` (avoiding date filters entirely). But this patch still handles existing installs that had the old filter-based cards.

## `fix_old_field_refs.py`

### Problem
Workspace shortcuts were referencing an old field name `custom_toc_generated` (removed in v1.0) instead of `custom_toc_recorded_by`. Number Cards had the same stale reference.

### Fix (two strategies)

**Strategy 1 — Workspace**: Delete the entire `TOC Buffer Management` workspace from DB so Frappe resyncs it from the corrected `toc_buffer_management.json` file during the same migrate run.
- Falls back to a direct SQL UPDATE on `tabWorkspace Shortcut` if delete fails.

**Strategy 2 — Number Cards**: For all Number Cards in module `Chaizup Toc`:
- Replaces `"custom_toc_generated"` → `"custom_toc_recorded_by"` in `filters_json`.
- Replaces `'"=",1'` → `'"=","By System"'`.
- Also applies the date filter fix from `fix_date_filters.py`.

## `fix_workspace_icon.py`

### Problem
The workspace icon was set to an SVG file path (e.g., `/assets/.../icon.svg`) which doesn't render in the Frappe sidebar — it only accepts standard icon names.

### Fix
If the workspace `TOC Buffer Management` has an icon containing `/` or `.svg`, replace it with `"graph"` (a valid Frappe octicon name).
