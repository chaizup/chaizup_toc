What Is This System?

This app replaces ERPNext's default "Auto Material Request" with Theory of Constraints (TOC) Buffer Management.

The one-line version: Instead of guessing how much to make/buy (forecasting), TOC watches what was actually consumed and replenishes exactly that amount. Every day. Automatically.

When you install this app, the default ERPNext reorder system is automatically disabled. The TOC engine takes over completely.

Read this first. After this, every screen in the app will make sense.

RLTReplenishment Lead TimeTotal DAYS from "we need more" until stock is on the shelf ready. Includes ALL steps.FG: Blend + Pack + QC + Move = 3 days. RM Tea Dust: PO to receipt = 10 days.

SR%Stock Remaining %How FULL the buffer is. Opposite of BP%. They always add to 100%. Like calling a glass half-empty vs half-full.Same example: SR% = 22.2%. Only 22% left in tank.

TMRToo Much RedBuffer too small — stock keeps falling into Red zone. The system increases the buffer to fix this.Target 900 × 1.33 = new Target 1,197.
TMGToo Much GreenBuffer too large — stock is always Green. Cash is locked up unnecessarily. System shrinks the buffer cautiously.Target 9,750 × 0.67 = new Target 6,533.
DAFDemand Adjustment FactorTemporary multiplier for KNOWN events: festivals, promotions, seasons. You set it manually.Diwali = 1.6× (buffer +60%). Summer = 0.7× (buffer −30%).

RMRaw MaterialsIngredients you PURCHASE from suppliers.Tea dust, sugar, milk powder, cardamom, ginger extract.
PMPackaging MaterialsMaterials that wrap or seal the final product. You PURCHASE these.Printed pouches, corrugated cartons, labels, tape.

First-Time Setup — Step by Step

Step 1: Install the chaizup_toc app → default auto Material Request is DISABLED automatically
Step 2: Open any Item → click the "TOC Setting" tab at the top of the form
Step 3: Check "Enable TOC"
Step 4: Select Buffer Type: FG / SFG / RM / PM
Step 5: Choose ONE: "Auto Purchase TOC" (for items you BUY) OR "Auto Manufacturing TOC" (for items you MAKE)
Step 6: ADU: Either check "Custom ADU" and type your value manually, OR select a Period and let the system auto-calculate daily
Step 7: (FG items only) Fill T/CU fields: Selling Price, TVC, Constraint Speed → T/CU auto-calculates
Step 8: (FG/SFG items) Link Default BOM for component availability checking
Step 9: Add Buffer Rules: one row per warehouse → enter RLT and VF → Target auto-calculates
Step 10: Save → Done! System runs automatically every day at 7:00 AM.

Daily Automatic Flow — What Happens Every Day

06:30 AM → ADU Auto-Update
  Calculates Average Daily Usage for every TOC-enabled item
  Reads: Delivery Notes (FG) / Stock Entries (RM/PM/SFG)
  SKIPS items where "Custom ADU" is checked

07:00 AM → Production Priority Run
  For EVERY TOC item: calculates IP (F2) → BP% (F3) → Zone → Order Qty (F4)
  Sorts ALL items: highest BP% first, T/CU (F5) as tie-breaker
  Creates Material Requests for Yellow + Red + Black zone items
  Tags each MR: "Recorded By = By System", Zone, BP%

07:30 AM → Procurement Alert
  Checks RM + PM buffers specifically
  Emails Purchase Managers if any item is in Red or Black zone

08:00 AM → Buffer Snapshot
  Logs every item's buffer state to TOC Buffer Log
  This data is used by DBM engine for weekly TMR/TMG analysis

Sunday 08:00 AM → DBM Weekly Check
  TMR: If item spent > 20% of RLT in Red → buffer increases by 33%
  TMG: If item stayed Green for 3 full cycles → buffer decreases by 33%

How Documents Connect — The Full Picture

Material Request — How TOC Creates It

TOC Engine runs at 7:00 AM
→ Calculates BP% for every enabled Item
→ Is BP% ≥ 33%? (Yellow or worse)
→ YES: Creates Material Request
   • Type = "Purchase" (if Auto Purchase TOC is checked on Item)
   • Type = "Manufacture" (if Auto Manufacturing TOC is checked on Item)
   • Qty = Target Buffer − Inventory Position (Formula F4)
   • "Recorded By [TOC App]" = "By System"
   • Zone, BP%, Target, IP all recorded on the MR
→ NO: Skip this item. Buffer is healthy (Green zone).
Where to see TOC-created MRs: Material Request list → filter "Recorded By = By System"
"Recorded By" field explained: "By System" = auto-created by TOC engine. "By User" = manually created by a human.

Work Order — How It Connects

Material Request (Type: Manufacture) is created by TOC
→ User opens the MR → clicks "Make → Work Order"
→ Work Order created with quantity from the MR
→ Production team starts manufacturing on VFFS
→ Stock Entry (type: Manufacture) completes the WO
→ Bin.actual_qty INCREASES for the FG item
→ Next day: TOC recalculates → BP% drops → zone improves

Purchase Order — How It Connects

Material Request (Type: Purchase) is created by TOC
→ User opens the MR → clicks "Make → Purchase Order"
→ PO sent to supplier → supplier delivers goods
→ Purchase Receipt received → Bin.actual_qty INCREASES for RM/PM
→ Next day: TOC recalculates → BP% drops → zone improves

ADU — Where It Calculates, How It Works

WHERE: Item → "TOC Setting" tab → Section 2 "Average Daily Usage"
WHEN: Every day at 6:30 AM by the scheduler
FORMULA: Total qty consumed in period ÷ Number of days in period
Item TypeWhat Data Is ReadERPNext Source
FG (Finished Goods)How many units were shipped to customersDelivery Note Item → qty (submitted docs only)
RM (Raw Materials)How many units were consumed in productionStock Entry Detail → qty (Material Issue / Manufacture types)
PM (Packaging)Same as RM — consumed in productionStock Entry Detail → qty
SFG (Semi-Finished)How many SFG units were consumed when packing FGStock Entry Detail → qty (where SFG was the source)
"Custom ADU" checkbox: If you check "Custom ADU (Manual Entry)" on the Item, the 6:30 AM scheduler will SKIP that item entirely. Your manually-entered ADU value is never overwritten. Use this when there is no consumption history in the system yet.

BOM and Multi-Level SFG

One FG can have a BOM with multiple SFGs + multiple RMs + PMs. Each SFG can itself have a BOM with more components. The system walks the full BOM tree (up to 5 levels deep).

FG BOM: Masala Tea 1kg
├── SFG: Masala Premix Blend (0.95 kg)
│   ├── RM: Tea Dust CTC (0.6 kg)
│   ├── RM: Sugar (0.2 kg)
│   └── RM: Milk Powder (0.15 kg)
├── RM: Tea Dust CTC (0.05 kg)
└── PM: 1kg Printed Pouch (1 pc)

Setup: Item → TOC Setting → Section 4 → link Default BOM → check "Check BOM Availability". Click the "Check BOM" button under the TOC button group to see component availability.

DBM — Dynamic Buffer Management

TriggerConditionWhat HappensSpeed
TMR (Too Much Red)Days in Red zone > 20% of RLTBuffer × 1.33 (increase 33%)FAST — stockout protection is urgent
TMG (Too Much Green)Green for 3 complete RLT cyclesBuffer × 0.67 (decrease 33%)SLOW — reduce excess cautiously
Safeguards built in: Maximum 3 consecutive TMR increases (then manual review needed). Buffer never shrinks below minimum floor (default 50 units). The asymmetry is deliberate: cost of stockout is far greater than cost of holding extra stock.

Frequently Asked Questions

Where does ADU auto-calculate?

Item → "TOC Setting" tab → Section 2. The scheduler runs at 6:30 AM daily. The result is written to "ADU Value [TOC App]" on the Item.

What if I just started and have no transaction history?

Check "Custom ADU" on the Item and type your estimated daily usage manually. The scheduler will not overwrite your value. Once you have 30+ days of real data, uncheck it to switch to auto-calculation.

Can an item be both purchased AND manufactured?

No. You must choose ONE: "Auto Purchase TOC" (for items you buy from suppliers) OR "Auto Manufacturing TOC" (for items you produce in-house). The system enforces this — you cannot check both.

What does "Recorded By" mean on Material Request and Work Order?

"By System" = auto-created by the TOC engine at 7:00 AM based on buffer calculations. "By User" = manually created by a person. System-generated MRs follow buffer priority sorting.

How do I test if everything works?

Go to TOC Dashboard → Actions → "🧪 Create Demo Data" (Administrator only). This creates 7 test items across all scenarios. Click "🗑️ Delete Demo Data" to clean up afterwards.

What happens to the default ERPNext auto reorder?

It is automatically disabled when this app is installed. The TOC engine completely replaces it. If you uninstall the app, the default reorder is re-enabled as a safety net.

What is the keyboard shortcut?

Ctrl+Shift+T from anywhere in ERPNext opens the TOC workspace instantly.
