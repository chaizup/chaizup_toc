# doctype/ — Custom DocTypes

All DocTypes in the chaizup_toc app. Includes standalone DocTypes (Buffer Log), child tables (Item Buffer, Warehouse Rule, Item Group Rule), and a singleton config DocType (TOC Settings).

```
doctype/
├── toc_buffer_log/           ← Standard DocType: daily buffer snapshot archive
├── toc_item_buffer/          ← Child Table (istable=1): per-warehouse buffer rule on Item
├── toc_settings/             ← Single DocType (issingle=1): app-wide configuration
├── toc_warehouse_rule/       ← Child Table (istable=1): warehouse classification in TOC Settings
└── toc_item_group_rule/      ← Child Table (istable=1): item group → buffer type mapping in TOC Settings
```

---

## DocType Summary

| DocType | Type | Parent | Purpose |
|---------|------|--------|---------|
| TOC Buffer Log | Standard | — | One row per item+warehouse+date. Historical archive for trend analysis and DBM. |
| TOC Item Buffer | Child Table | Item (`custom_toc_buffer_rules`) | ADU/RLT/VF/DAF parameters per warehouse. Drives all F1-F6 calculations. |
| TOC Settings | Singleton | — | Zone thresholds, DBM params, DAF, warehouse rules, item group rules. |
| TOC Warehouse Rule | Child Table | TOC Settings (`warehouse_rules`) | Classify warehouses as Inventory/WIP/Excluded for F2 calculation. |
| TOC Item Group Rule | Child Table | TOC Settings (`item_group_rules`) | Auto-assign buffer type (FG/SFG/RM/PM) from Item Group hierarchy. |

## Custom Fields on Built-in DocTypes

These are not separate DocTypes — they are `Custom Field` records installed via `setup/install.py → _install_custom_fields()`. They appear as additional fields on existing ERPNext DocTypes.

### Item — TOC Settings Tab

| Field Label | Fieldname | Type | Purpose |
|-------------|-----------|------|---------|
| Enable TOC | `custom_toc_enabled` | Check | Master switch for this item in TOC system |
| Buffer Type | `custom_toc_buffer_type` | Select (FG/SFG/RM/PM) | Manual type override (auto-resolved from item group rules if blank) |
| ADU Value | `custom_toc_adu_value` | Float | Computed ADU (from delivery notes or manual) |
| ADU Last Updated | `custom_toc_adu_last_updated` | Datetime | When ADU was last auto-calculated |
| ADU Period (days) | `custom_toc_adu_period` | Int | Lookback period used for ADU calculation |
| Custom ADU | `custom_toc_custom_adu` | Check | If checked, ADU is manually set (not auto-calculated) |
| Selling Price | `custom_toc_selling_price` | Currency | For T/CU calculation |
| TVC | `custom_toc_tvc` | Currency | Totally Variable Cost per unit |
| Constraint Speed | `custom_toc_constraint_speed` | Float | Units per minute on bottleneck machine |
| T/CU | `custom_toc_tcu` | Currency | Throughput per Constraint Unit (auto-calculated) |
| Default BOM | `custom_toc_default_bom` | Link → BOM | BOM to use for kitting check |
| Check BOM Availability | `custom_toc_check_bom_availability` | Check | If checked, BOM component check runs in real-time |
| Auto Purchase | `custom_toc_auto_purchase` | Check | This item generates Purchase MRs (mutually exclusive with Auto Manufacture) |
| Auto Manufacture | `custom_toc_auto_manufacture` | Check | This item generates Work Orders (mutually exclusive with Auto Purchase) |
| TOC Buffer Rules | `custom_toc_buffer_rules` | Table → TOC Item Buffer | Per-warehouse buffer configuration rows |

## Relationship Diagram

```
Item (built-in ERPNext)
├── custom_toc_enabled: Check
├── custom_toc_buffer_type: Select (FG/SFG/RM/PM) — manual or auto-resolved
├── custom_toc_adu_value: Float — auto-updated by daily ADU task
├── custom_toc_tcu: Currency — (Price − TVC) × Constraint Speed
└── custom_toc_buffer_rules: Table
    └── TOC Item Buffer (child) ← one row per warehouse this item should be buffered at
        ├── warehouse, adu, rlt, variability_factor
        ├── target_buffer (F1: ADU × RLT × VF) — auto-calculated
        ├── daf, adjusted_buffer (F6: Target × DAF) — auto-calculated
        └── tmr_count, last_dbm_date — DBM tracking

TOC Buffer Log (standalone)
├── item_code → Item
├── warehouse → Warehouse
├── log_date: Date (indexed)
├── zone, bp_pct, ip, target (F2/F3 snapshot)
└── mr_created → Material Request (if MR was auto-generated)

TOC Settings (singleton)
├── red_zone_threshold: 67 (default)
├── yellow_zone_threshold: 33 (default)
├── auto_generate_mr: Check
├── mr_zones: Select
├── enable_dbm: Check
├── DBM params (tmr_red_pct_of_rlt, tmg_cycles_required, dbm_adjustment_pct, ...)
├── default_daf: Float (global DAF, 1.0 = normal)
├── warehouse_rules: Table
│   └── TOC Warehouse Rule (child)
│       └── warehouse, warehouse_purpose (Inventory/WIP/Excluded)
└── item_group_rules: Table
    └── TOC Item Group Rule (child)
        └── item_group, buffer_type, include_sub_groups, priority

Material Request (built-in ERPNext + custom fields)
└── custom_toc_recorded_by: "By System"
└── custom_toc_zone, custom_toc_bp_pct (snapshot at creation time)

Work Order (built-in ERPNext + custom fields)
└── custom_toc_recorded_by: "By System" (for auto-generated WOs from kitting)
└── custom_toc_zone, custom_toc_bp_pct
```

## Fixture Management

All custom fields use `module = "Chaizup Toc"`. Tracked as fixtures in `hooks.py`:

```python
fixtures = [
    {"doctype": "Custom Field", "filters": [["module", "=", "Chaizup Toc"]]},
    {"doctype": "Property Setter", "filters": [["module", "=", "Chaizup Toc"]]},
]
```

**Exporting after changes:**
```bash
bench --site your-site export-fixtures --app chaizup_toc
# Writes to: apps/chaizup_toc/chaizup_toc/fixtures/custom_field.json
#            apps/chaizup_toc/chaizup_toc/fixtures/property_setter.json
```

**Importing on new site:**
```bash
bench --site new-site migrate
# Frappe reads fixtures/ folder and creates all Custom Field records
```

## How DocTypes Are Registered

Frappe scans the `doctype/` subfolder during `bench migrate` and creates/updates all DocType records based on their JSON files. The file structure is:

```
doctype/toc_buffer_log/
├── toc_buffer_log.json   ← Field definitions, permissions, autoname, is_submittable
└── toc_buffer_log.py     ← Python controller (may be empty — `class TOCBufferLog(Document): pass`)
```

The JSON file is authoritative. The Python controller handles business logic. Both are version-controlled.

## DocType Access Patterns

### TOC Buffer Log (Standard)

```python
# Read (list query with filters)
logs = frappe.get_all("TOC Buffer Log",
    filters={"item_code": "FG-MASALA-1KG", "log_date": today()},
    fields=["zone", "bp_pct", "inventory_position", "mr_created"])

# Read (direct SQL for performance)
frappe.db.sql("""
    SELECT log_date, zone, bp_pct
    FROM `tabTOC Buffer Log`
    WHERE item_code = %s AND log_date >= %s
    ORDER BY log_date DESC
""", (item_code, from_date), as_dict=True)

# Write (new record)
log = frappe.new_doc("TOC Buffer Log")
log.item_code = "FG-MASALA-1KG"
log.log_date = today()
log.zone = "Red"
log.flags.ignore_permissions = True
log.insert()
```

### TOC Item Buffer (Child Table)

```python
# Access via parent Item
item = frappe.get_doc("Item", "FG-MASALA-1KG")
for rule in item.get("custom_toc_buffer_rules") or []:
    print(rule.warehouse, rule.adu, rule.target_buffer)

# Direct query (faster for batch)
rules = frappe.get_all("TOC Item Buffer",
    filters={"parent": "FG-MASALA-1KG", "parentfield": "custom_toc_buffer_rules", "enabled": 1},
    fields=["name", "warehouse", "adu", "rlt", "variability_factor",
            "target_buffer", "adjusted_buffer", "daf", "tmr_count"])

# DBM update (bypasses validate — intentional)
frappe.db.set_value("TOC Item Buffer", rule_name, {
    "target_buffer": new_target,
    "tmr_count": new_count,
    "last_dbm_date": today(),
})
frappe.db.commit()
```

### TOC Settings (Singleton)

```python
# Read (cached — no DB hit if already loaded in this request)
settings = frappe.get_cached_doc("TOC Settings")
threshold = settings.red_zone_threshold        # 67
enable_dbm = cint(settings.enable_dbm)        # 1
default_daf = flt(settings.default_daf)       # 1.0

# Read single value
daf = frappe.db.get_single_value("TOC Settings", "default_daf")

# Write (bypasses cache)
frappe.db.set_single_value("TOC Settings", "default_daf", 1.6)
frappe.db.set_single_value("TOC Settings", "daf_event_name", "Diwali 2026")
frappe.db.commit()
# Note: clear cache after write if other code reads cached value same request
```
