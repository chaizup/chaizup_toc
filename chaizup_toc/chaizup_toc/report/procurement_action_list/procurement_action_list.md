# `procurement_action_list/` — Procurement Action List Report

## Role
Purchase-focused daily report for RM and PM items. Procurement team uses this to determine which materials to order and at what urgency (standard vs. express shipping).

Type: **Script Report**

## Filters (`procurement_action_list.js`)

| Filter | Type |
|--------|------|
| `company` | Link (defaults to user company) |
| `warehouse` | Link |
| `zone` | Select (Green/Yellow/Red/Black) |
| `item_code` | Link→Item |

## Data Source
`calculate_all_buffers(buffer_type="RM")` + `calculate_all_buffers(buffer_type="PM")`, merged and sorted by `bp_pct DESC`.

## Columns

| Column | Description |
|--------|-------------|
| Rank | Sequential |
| Material | Item Code |
| Name | Item Name |
| Type | RM or PM |
| Target (F1) | ADU × RLT × VF |
| On-Hand | `actual_qty` |
| On-Order | `ordered_qty` (open POs) |
| Committed | `reserved_qty` |
| IP (F2) | On-Hand + On-Order − Committed |
| BP% (F3) | Buffer Penetration % |
| Zone | Green/Yellow/Red/Black |
| PO Qty (F4) | Target − IP |
| Freight | Recommendation by zone |
| Action | Zone action text |

## Freight Recommendation
| Zone | Freight Mode |
|------|-------------|
| Green | N/A |
| Yellow | Standard |
| Red | Express/Air |
| Black | EMERGENCY |

## Row Formatting
Zone column: colored pill (same colors as Production Priority Board).

## Summary
| Card | Value |
|------|-------|
| Total | all RM/PM buffers |
| Urgent | Red/Black count |

## Usage
- Run daily at 7:35 AM (after procurement run).
- Procurement officer creates Purchase Orders for Red/Black items immediately.
- Express freight is triggered automatically for Red zone items via this recommendation.
- Yellow items are standard POs with normal lead time.

## Note
This report is READ-ONLY. There is no "Generate MRs" button here. Purchase MRs are generated by `daily_production_run()` at 07:00 AM (runs for all buffer types). This report is for human decision-making on urgency and freight.
