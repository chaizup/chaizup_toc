# `overrides/` — ERPNext DocType Overrides

## Purpose
Hooks into built-in ERPNext DocType lifecycle events to enforce TOC rules. No new DocTypes — only modifies behavior of existing ones.

## Files

| File | Overrides |
|------|-----------|
| `item.py` | Item validate — ADU validation, T/CU calc, BOM check |
| `material_request.py` | MR validate — TOC compliance warning |
| `reorder_override.py` | Replaces `erpnext.stock.reorder_item.reorder_item()` |

---

## `item.py`

### `on_item_validate(doc, method)`
Fires via `hooks.py → doc_events["Item"]["validate"]`.

Runs only when `doc.custom_toc_enabled = 1`. Performs:

1. **R1 — ADU validation**: If `custom_toc_custom_adu` is checked but `custom_toc_adu_value <= 0`, shows an orange alert (non-blocking).

2. **R2 — Mutual exclusion**: If both `auto_purchase` and `auto_manufacture` are checked simultaneously → throws an error. An item cannot be both purchased AND manufactured.

3. **Monitor-only warning**: If neither auto_purchase nor auto_manufacture is checked, shows an orange alert: "TOC will monitor this item but will NOT auto-create Material Requests."

4. **F5 — T/CU calculation** (FG items only):
   ```
   T/CU = (selling_price - tvc) × constraint_speed
   ```
   Sets `custom_toc_tcu`. If price or speed is 0, sets T/CU to 0.

5. **R3 — BOM link validation**: If `custom_toc_default_bom` is set, verifies the BOM's `item` field matches the current Item. Throws if mismatched.

6. **Buffer rules existence check**: If TOC is enabled but `custom_toc_buffer_rules` is empty, shows an orange alert.

## `material_request.py`

### `validate_toc_compliance(doc, method)`
Fires via `hooks.py → doc_events["Material Request"]["validate"]`.

- **No-op**: If `custom_toc_recorded_by == "By System"` (auto-generated by TOC), returns immediately.
- **Warning only** (non-blocking): If any item in the MR is TOC-managed (`custom_toc_enabled=1`), shows an orange informational message listing those items.

Message content:
> "The TOC system auto-generates Material Requests daily at 7:00 AM based on Buffer Penetration %. Manual MRs may conflict with TOC calculations."

This does NOT block the save — it's informational only.

## `reorder_override.py`

### `toc_reorder_item()`
Registered in `hooks.py → override_whitelisted_methods`:
```python
"erpnext.stock.reorder_item.reorder_item": "chaizup_toc.overrides.reorder_override.toc_reorder_item"
```

This is a belt-and-suspenders safeguard. Even if someone re-enables "auto_indent" in ERPNext's Stock Settings, this function intercepts the call and:

1. Logs a WARNING explaining why the default was intercepted.
2. Auto-disables `Stock Settings.auto_indent` again.
3. Returns `None` (same contract as the original function).
4. Does NOT generate any Material Requests via default ERPNext logic.

## Bug History

### ~~BUG-CRITICAL (FIXED): Missing `on_buffer_rule_validate` function~~

`hooks.py` previously registered a doc_event:
```python
"TOC Item Buffer": {
    "validate": "chaizup_toc.overrides.item.on_buffer_rule_validate",
},
```

This function did not exist in `overrides/item.py`, causing every Item save with buffer rules to fail with an `AttributeError`.

**Fixed**: The entire `"TOC Item Buffer"` entry was removed from `doc_events` in `hooks.py`. All buffer calculations are handled by the `TOCItemBuffer.validate()` controller in `chaizup_toc/doctype/toc_item_buffer/toc_item_buffer.py`.
